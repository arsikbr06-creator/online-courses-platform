## Слайд 4: Архитектура системы

```
                  NGINX (Балансировщик)
                          ↓
        ┌─────────────────┼─────────────────┐
        ↓                 ↓                 ↓
    Frontend          Backend          Backend
    (React)        (Node.js x3)      (Node.js x3)
        ↓                 ↓                 ↓
        └─────────────────┼─────────────────┘
                          ↓
              ┌───────────┴───────────┐
              ↓                       ↓
        PostgreSQL                 Redis
        (+ реплики)               (Кэш)
              ↓
        Prometheus + Grafana
        (Мониторинг)
```

**7 контейнеров**: Frontend, Backend (x3), PostgreSQL, Redis, NGINX, Prometheus, Grafana

## Слайд 6: Кластерная архитектура

### Docker Swarm конфигурация:
- **Manager узлы**: 3 (для кворума)
- **Worker узлы**: 5+
- **Репликация сервисов**:
  - Backend API: 3 реплики
  - Frontend: 2 реплики
  - PostgreSQL: 1 master + 2 read replicas
  - Redis: 1 master + 2 replicas

### Преимущества:
✓ Высокая доступность  
✓ Автоматическое восстановление  
✓ Rolling updates без простоя  
✓ Балансировка нагрузки

## Слайд 7: Сетевое взаимодействие

### Overlay сети:
1. **frontend-network**
   - Соединяет NGINX ↔ Frontend
   - Шифрование трафика

2. **backend-network**
   - Соединяет Backend ↔ БД ↔ Redis
   - Изолирована от frontend (безопасность)

3. **monitoring-network**
   - Prometheus ↔ все сервисы
   - Сбор метрик

### Особенности:
- DNS-разрешение имен сервисов
- Встроенная балансировка (routing mesh)
- Изоляция компонентов

## Слайд 8: CI/CD Pipeline

### GitHub Actions workflow:

```
Commit → Lint → Tests → Build → Security Scan
                          ↓
                    Docker Registry
                          ↓
              ┌───────────┴───────────┐
              ↓                       ↓
          Staging               Production
        (auto deploy)         (manual approve)
              ↓                       ↓
        Smoke tests              Smoke tests
```

### Компоненты:
- **Автоматизация**: сборка, тестирование, деплой
- **Стратегия**: Blue-Green deployment
- **Безопасность**: Snyk, Trivy сканирование
- **Мониторинг**: автоматические уведомления
